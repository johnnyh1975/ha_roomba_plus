## 🐛 Fixed: onboarding failed entirely for some V4/Prime robots

A field report (GitHub issue, darealgugu) found that a Roomba Plus 505 Combo (SKU `N185240`)
was correctly identified by the cloud login step, but then failed onboarding with a "not found
on local network" error — even with the correct IP entered manually. That's expected: this is a
cloud-only robot with no local network protocol at all, so a local-network search can never
succeed for it.

The cause: only one specific SKU prefix (`G185020`, Roomba Plus 405 Combo) was recognized as
"cloud-only V4/Prime" — every other robot on the account fell through to the Classic setup path,
which always tries local network completion.

Fixed: SKU-prefix recognition is now an extensible set (`G` and `N` confirmed so far, both
"Combo"-family cloud-only robots — `G185020`/Roomba Plus 405 Combo, `N185240`/Roomba Plus 505
Combo) instead of a single hardcoded check. If your V4/Prime robot still fails onboarding after
updating, please open an issue with the SKU code and the "V4/Prime" vs "Classic" label the
robot-picker screen shows for it — that's exactly the information needed to add it.

4,038 tests passing, no other changes in this release.
